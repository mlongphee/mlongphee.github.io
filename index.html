Keele CC

Built an app for Parents/Clients to stay connected and updated on camps, programs, schedules, and events using Andriod SDK and Firebase.

Upon creating an account, another account activation is required which provides a layered security system allowing people we only want on the app.

A user will be given a client role by default which allows users to view information on the app. Staff are given an admin role which grants staff the ability to post, delete posts, change camp information and grant access to people to use the app.

Posts from the different camps show up among the newsfeed, events, what’s up, and lost & found sections of the app. When someone posts among one of the sections, you are notified through a notification system which can be enabled/disabled for each section. Users are also able to filter through camps and programs allowing them to easily and quickly get the information they want.

Designed and developed the android app. 60+ users used the app when it was released.

Chess AI

Designed an AI to play Chess at ~1500 ELO @ 15s. Implemented minMax and alpha-beta pruning algorithms to reduce the amount of positions evaluated making the AI more efficient. Completed in Javascript using the ChessboradJS library for the UI.

The first thing that was done to implement the chess AI was a minimax search and link it, to the GUI. It was nothing out of the ordinary, a simple state space search. However, it quickly evolved to using Alpha-Beta pruning, which optimizes the minimax search algorithm by cutting some nodes that should not be expanded because there is a better move already found.

The next step was to add in some machine learning. The first attempt was to implement a Monte Carlo Reinforcement learning algorithm alongside with a Monte Carlo tree search. However, despite managing to implement this, the AI ran into time & efficiency issues.

Canadian Political Sentiment Analysis via Twitter

A single misquote or negative news about a candidate can be the difference between him/her winning or losing the election. It becomes key to have a platform that enables political parties to make data driven decisions to guide and direct the candidate’s campaign.

With a team of 2 people, we developed a platform that analyzes public sentiment of the political parties in Canada. One of the main things we specifically focus on finding is the public sentiment of each political party for all of Canada, its provinces and its ridings.

Furthermore, we used twitter data to help us answer some other useful questions such as:
Location Analysis: What is the political sentiment of major geo-locations in Canada? (ex. Provinces and Ridings)
Real Time Analysis: What is the current sentiment of each province in Canada?
Candidate Analysis: What is the political sentiment towards major politicians of each party?
Value Analysis: What do people value in each province? (ex. Taxes, Economy, Education)

Throughout the project we use some of the industry’s leading tool such as Apache spark, Spark Streaming, HDFS and MapReduce which helped us develop an efficient, distributed and scalable project. Furthermore, using Sparks machine learning library we can develop a deep learning model to predict future election results using the data from our sentiment analysis.

Kijiji Scraper/Notifier

Created a near real time Kijiji post notifier where when given a Kijiji URL it scrapes the page on a set time interval and notifies the user when there is a new post.

Using a Kijiji scraper, it scrapes the first 10 posts on the given Kijiji URL and saves them in a JSON file. On the next scrape of the URL, it compares the current results with the previously saved JSON results and checks to see if there is a new post. If there are new posts, it then notifies me through email of the new posts.

By running the script on AWS Lambda, it allows me to continuously run the script throughout the day without any downtime. The script is run on a set time interval of 2 minutes, but can be changed if needed.

Developed on Node.js, Javascript and AWS Lambda.

Rooster Graphics

Designed and developed a raster graphics editor that allows users to create and edit images interactively on the editor and save them in a JPEG format.

Rooster graphics editor went through 2 design phases on which we revised and finalized our program design focusing on fundamental design principles.

CopOrDrop

Designed and developed a social app that allows users to rate newly released shoes while keeping up to date with what new shoes have been released.

Posts are given a “trending score”, which is determined by the most upvotes and the posts age. The “popular” section displays recently most trending shoes at the moment. The newer and more active the post, the higher it will appear on the list.

Users are able to create new posts, upvote or downvote posts, comment on posts, edit their profile and reply to comments in the comment section. Comments are laid out through a nesting system, where replies are nested under their parents comment.

Users are able to also customize their profile by uploading a profile picture and cover photo, give themselves a short description, and add addition information such as favorite shoes.
